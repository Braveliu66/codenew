# 系统需求说明

## 1. 项目定位

本系统是面向非商业研究和毕业设计场景的 3D Gaussian Splatting 重建平台。系统提供从图片、视频或实时摄像头输入，到极速预览、精细重建、Web 查看、导出、分享和后台管理的一体化流程。

系统不要求用户理解底层算法。用户只表达操作意图，例如上传素材、生成预览、开始精细重建、导出 Mesh 或分享结果，后端负责自动选择固定算法管线。

## 2. 用户角色

| 角色 | 主要目标 | 典型操作 |
| --- | --- | --- |
| 普通用户 | 上传素材并查看重建结果 | 登录、新建项目、上传图片/视频、实时拍摄、查看预览、补传素材、发起精细重建、导出文件、分享项目、反馈问题 |
| 管理员 | 管理系统资源、用户和任务 | 查看 GPU 状态、队列深度、任务日志、每个用户的项目数、存储占用和训练占用 |
| 研究维护者 | 管理算法版本和合规信息 | 登记算法仓库、许可证、commit hash、权重来源 |

## 3. 功能需求

### 3.1 项目管理

- 用户可以创建、查看、删除项目。
- 项目应记录名称、标签、输入类型、状态、创建时间、更新时间、错误信息、大小统计和主要产物。
- 项目支持图片、视频和实时摄像头三类输入来源。
- 项目管理页面以卡片形式展示项目效果图、名称、时间、标签、状态、大小和进度。
- 项目支持按名称搜索，按时间、标签、训练状态筛选。
- 训练中的项目若没有 3DGS 结果，应使用上传数据中的一帧或一张图片作为卡片预览图。
- 项目详情页支持查看、删除、导出 PLY、分享和全屏渲染。

### 3.2 素材上传

- 图片上传支持多文件和分片上传。
- 视频上传支持大文件上传和断点续传。
- 上传完成后，系统将原始素材保存到对象存储，并触发预处理任务。
- 原始素材应按用户和项目隔离存储。
- 上传页面支持编辑项目名称和标签。
- 用户可以继续补传图片或视频，也可以删除已上传的部分素材。
- 图片以缩略图展示，点击缩略图可以查看大图。
- 上传页面展示图片数量、视频大小、总占用、分辨率、时长等统计数据。

### 3.3 极速预览

- 图片输入使用 LiteVGGT + EDGS 管线生成 `preview.spz` 和低细节 LOD 文件。
- 视频输入使用 LingBot-Map 生成粗略点云并转换为 Spark-SPZ；实时摄像头输入使用 LingBot-Map streaming 分片生成增量 SPZ segment。
- 预览任务优先级高于精细重建任务。
- 预览完成后，前端应能立即加载可交互模型。
- 预览运行时前端应展示当前阶段、进度条、预计剩余时间。
- 预览完成后，系统应给出素材质量提示，例如建议补拍的方向、缺失视角或模糊素材提醒。

### 3.4 精细重建

- 用户可以在预览完成后发起精细重建。
- 系统根据素材质量自动判断是否启用模糊增强、稀疏视角初始化或长视频全局优化。
- 精细重建产出 `final.ply`、`final_web.spz`、多级 `.rad` 文件和 `metrics.json`。
- 长视频全局优化为可选能力，默认不强制启用。
- 精细重建运行时前端应展示阶段、进度条和预计剩余时间。
- 精细重建完成后，项目详情页默认展示精细重建模型；若尚未完成，则展示极速预览模型、上传素材封面图或训练中状态。

### 3.5 实时摄像头重建

- 用户可以选择实时摄像头输入创建项目。
- 页面左侧显示摄像头画面，右侧实时显示粗重建效果。
- 录制期间展示录制时长、帧率、数据大小和实时重建状态。
- 用户结束录制后，可以选择重拍或进入精细重建。
- 进入精细重建后，系统使用录制素材生成更高清的 3DGS 模型。

### 3.6 LOD 与 Web 查看

- 系统生成 3 到 4 个 LOD 级别。
- Viewer 初始加载低细节层，再逐步加载高清层。
- 桌面和移动设备应根据性能自动选择合适 LOD。
- 所有 3DGS 渲染区域都支持全屏。
- Viewer 优先保证帧率，帧率稳定后自动提高清晰度。

### 3.7 导出与分享

- 用户可以导出 PLY、OBJ、GLB 等格式。
- 导出文件通过签名 URL 下载，默认有效期 1 小时。
- 分享页面应展示可交互的 Web 模型，不暴露内部存储路径。

### 3.8 用户账号与反馈

- 系统支持用户登录。
- 用户可以查看项目总览，包括项目总数、训练中项目数、已完成项目数、总占用空间。
- 用户可以提交问题反馈，反馈内容应记录项目关联、文字描述、截图或附件。

### 3.9 管理面板

- 管理员可以查看任务队列、GPU 负载、任务耗时、失败原因和用户存储占用。
- 管理面板应支持查看 Worker 心跳和最近任务日志。
- 管理员可以按用户查看项目数、存储占用、训练占用、正在运行任务和历史任务。

### 3.10 合规展示

- 系统“关于”页面必须列出所有集成算法的名称、许可证、仓库地址和版本信息。
- 非商业限制需要在页面中明确展示。
- 编写后端算法代码时必须接入真实 GitHub 算法仓库或本地真实算法实现，不能用占位函数、假文件或固定假结果冒充算法通过。

## 4. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 极速预览优先保证可交互结果尽快出现，精细重建优先保证质量 |
| 并发 | API 使用异步处理，任务通过 Redis 队列进入 GPU Worker |
| 可扩展 | GPU Worker 可横向扩展，调度器按显存、利用率、任务类型和优先级分配任务 |
| 稳定性 | 预览任务超时 5 分钟，精细重建任务超时 2 小时，Mesh 导出超时 30 分钟 |
| 可观测 | 记录任务状态、进度、日志、错误、产物路径和关键指标 |
| 存储 | 使用 MinIO 或 S3 保存原始素材、预览、最终模型、日志和导出包 |
| 安全 | 用户只能访问自己的项目和签名下载链接 |
| 合规 | 系统仅供非商业用途，算法许可证信息必须可审计 |
| 实现真实性 | 算法相关功能必须执行真实代码；未配置算法环境时应明确失败或返回不可用状态 |

## 5. 验收标准

1. 图片极速预览可以完成上传、入队、处理、生成产物、前端加载的完整闭环。
2. 视频极速预览可以生成可加载的预览模型，并在前端展示任务进度。
3. 实时摄像头粗重建可以周期性推送新的预览版本。
4. 精细重建可以生成 `final.ply`、`final_web.spz`、LOD 文件和 `metrics.json`。
5. Mesh 导出可以生成 `.ply`、`.obj`、`.glb` 文件，并提供下载链接。
6. Viewer 在桌面和移动端都能加载模型，并保持基本交互流畅。
7. 管理面板可以显示 GPU 任务、队列和用户统计。
8. 关于页面可以展示算法列表、许可证和原仓库链接。
9. 首页可以显示系统名称、CPU、GPU、显存占用，以及训练中项目的进度和预计剩余时间。
10. 实时视频页面可以同时显示摄像头画面和实时粗重建效果，结束后支持重拍或进入精细重建。
11. 算法功能不得依赖占位函数或假产物通过验收。



仍未满足验收标准第 4 条：真实 GPU 环境下还没有跑通 Faster-GS/Fused3DGS 训练，也未实际生成 `final.ply`、`final_web.spz`、`.rad` LOD 和 `metrics.json` 的完整成功链路。部署前必须接入真实训练命令、SPZ converter、RAD exporter，并通过 fine runtime preflight 和小样本端到端验收。

完整差距清单见 `11-fine-reconstruction-deployment-gap.md`。
